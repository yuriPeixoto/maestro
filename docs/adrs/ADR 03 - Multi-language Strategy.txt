===============================================================================
ADR 003: ESTRATÉGIA MULTILÍNGUE DE DOCUMENTAÇÃO
===============================================================================

STATUS: Proposto / Aceito
CONTEXTO: O projeto Maestro visa ser acessível globalmente como open source, mantendo ao mesmo tempo uma forte base de usuários e desenvolvedores lusófonos.

-------------------------------------------------------------------------------

1. DECISÃO: LÍNGUA PRIMÁRIA E SECUNDÁRIA
-----------------------------------------
- LÍNGUA PRIMÁRIA: Inglês (en-US)
- LÍNGUA SECUNDÁRIA: Português Brasileiro (pt-BR)

MOTIVAÇÃO: 
- O Inglês é o padrão de facto na comunidade de tecnologia e projetos open source globais.
- O Português Brasileiro é mantido para suportar a comunidade local e facilitar o onboarding de talentos regionais.

2. PADRÃO DE NOMENCLATURA DE ARQUIVOS
--------------------------------------
- O arquivo principal de documentação (ex: README.md) deve estar em Inglês.
- Traduções para Português Brasileiro devem seguir o sufixo `.pt-BR.md`.
- Exemplo: `README.md` (Inglês) e `README.pt-BR.md` (Português).

3. ESCOPO DA MULTILINGUAGEM
----------------------------
- Código-fonte (variáveis, funções, comentários internos): EXCLUSIVAMENTE em Inglês.
- Documentação de Usuário (README): Bilíngue (EN/PT-BR).
- ADRs e Documentação Técnica Interna: Podem ser iniciadas em PT-BR mas devem ser migradas ou traduzidas para EN conforme o projeto escala internacionalmente.

-------------------------------------------------------------------------------
NOTAS FINAIS:
Esta decisão garante que o projeto não possua barreiras para contribuições internacionais enquanto valoriza a origem do projeto.
===============================================================================
